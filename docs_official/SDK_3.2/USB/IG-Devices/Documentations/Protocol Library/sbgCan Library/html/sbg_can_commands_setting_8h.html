Initialize the kalman filter with a no motion assumption until a motion is detected for faster startup.

enum _SbgCanPowerModeDevice

Defines the device power mode options

Enumerator
SBG_CAN_DEVICE_MAX_PERF 

Max. performance mode. Allows reduced latency as well as heave computation

SBG_CAN_DEVICE_NORMAL 

Normal mode; Limited power consumption but we cannot get heave computation on this mode

enum _SbgCanPowerModeGps

Define the GPS power mode options

Enumerator
SBG_CAN_GPS_MAX_PERF 

Max. performance mode.

SBG_CAN_GPS_ECO_MODE_1 

Eco mode 1: Current peaks are limited;
startup might be longer than in MAX_PERF mode.
(Supported on GPS hardware V3 and above)

SBG_CAN_GPS_ECO_MODE_2 

Eco mode 2: Current peaks are limited;
In addition, once sufficient satellites are visible, the GPS enters into a low power mode and stops new satellites tracking
(Supported on GPS hardware V3 and above)

SBG_CAN_GPS_OFF_MODE 

GPS receiver is turned off.

SbgErrorCode sbgCanRestoreSettings ( SbgCanDeviceHandle  deviceHandle)

Restores all settings to factory defaults and save them to FLASH memory.
Calibration data such as gyroscopes bias and magnetometers calibration are not affected by this command!
The device bit rate is reseted to 1000 Mbit/s.
The device send the result of this command afterward with the default device bit rate.
This command change the bit rate on the device and the library.
To change only the bit rate used by the library, you have to use the function sbgCanBusChangeBitRate.

Parameters
[in]deviceHandleA valid ig can device handle.
Returns
SBG_NO_ERROR if no error.

SbgErrorCode sbgCanSetFrameId ( SbgCanDeviceHandle  deviceHandle,
SbgCanId  defaultMsgId,
SbgCanMessageFormat  newMsgFormat,
uint32  newMsgId 
)

For a specific CAN message, defines a custom id or disable the message using the specific id SBG_CAN_DISABLED_FRAME.
For example, to define SBG_CAN_OUTPUT_EULER CAN message id to 0x0CD3 use the following syntax:
sbgCanSetFrameId(deviceHandle, SBG_CAN_OUTPUT_EULER, 0x00000CD3);

Parameters
[in]deviceHandleA valid ig can device handle.
[in]defaultMsgIdId of the CAN message we would like to define (use an id from SbgCanId enum.)
[in]newMsgFormatNew CAN message format (SBG_CAN_ID_STANDARD or SBG_CAN_ID_EXTENDED).
[in]newMsgIdThe new CAN message id or SBG_CAN_DISABLED_FRAME to disable this message.
Returns
SBG_NO_ERROR if no error.

SbgErrorCode sbgCanSetProtocolMode ( SbgCanDeviceHandle  deviceHandle,
uint16  bitRate 
)

Defines the bit rate of the device CAN communication.
If the command is valid, the device acknoledge the bit rate change at the current speed and THEN change it's bit rate.
This command change the bit rate on the device and the library.
To change only the bit rate used by the library, you have to use the function sbgCanBusChangeBitRate.

Parameters
[in]deviceHandleA valid ig can device handle.
[in]bitRateThe new device bit rate to use in kBit/s.
Possible values are:
  • 1 000 kBit/s
  • 500 kBit/s
  • 250 kBit/s
  • 125 kBit/s
  • 100 kBit/s
  • 50 kBit/s
  • 20 kBit/s
  • 10 kBit/s
Returns
If SBG_NO_ERROR, the device has been configured to the new speed.
